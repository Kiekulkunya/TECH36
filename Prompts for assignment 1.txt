🔍 Prompt 1 — Data Profiling (Your Foundation)
I have a retail user behavior dataset with 108,584 rows and 18 columns. 
Here's the schema:

- session_id, user_id: identifiers
- timestamp_utc: event datetime (ISO format)
- event_index: step number within a session
- user_action: one of [view, click, wishlist, add_to_cart, purchase, drop]
- product_id, category (8 types), brand (33 brands)
- price: float, range $7–$500
- channel: mobile / web / app
- device_type: desktop / tablet / android / ios
- region: 9 countries (US, UK, JP, IN, AU, CA, BR, DE, FR)
- traffic_source: organic / direct / paid_search / social / email / affiliate / referral
- time_spent_sec, session_length, interaction_count: behavioral metrics
- is_conversion: binary (3.87% conversion rate)
- drop_off_flag: binary (12.7% drop-off rate)

The data tracks a user's journey through a retail site — each row is one event in a session.

Describe the analytical story hidden in this data. What are the 5 most interesting questions 
this dataset can answer for a business? What should I visualize first?

🏗️ Prompt 2 — The Main Dashboard Build
Build a polished, dark-themed admin dashboard in a single HTML file using D3.js and vanilla JS 
for this retail behavior dataset (108k rows, provided as inline JSON aggregates — do not load 
the raw CSV).

The dashboard must include:

HEADER: Title "RetailPulse Analytics", live KPI cards for:
  - Total Sessions, Unique Users, Conversion Rate (3.87%), Drop-off Rate (12.7%), Avg Price

CHART 1 — Funnel Chart (D3):
  Show the user action funnel: view → click → wishlist → add_to_cart → purchase → drop
  Show drop-off % between each stage. Use a horizontal funnel shape, not a bar chart.

CHART 2 — Heatmap (D3):
  Rows = 9 regions (US, UK, JP, IN, AU, CA, BR, DE, FR)
  Columns = 8 categories (Electronics, Groceries, Sports, Accessories, Beauty, Home, Shoes, Apparel)
  Color = conversion rate per cell. Tooltip shows exact rate.

CHART 3 — Stacked Bar Chart (D3):
  X-axis = traffic source (7 sources)
  Stacked = device type (desktop/tablet/android/ios)
  Y-axis = session count
  Show which traffic sources bring which devices

CHART 4 — Line Chart (D3):
  X-axis = hour of day (0–23), aggregated from timestamp_utc
  Y-axis = event volume
  3 lines: total events, conversions, drop-offs
  Show peak hours clearly

SIDEBAR: Category filter (checkboxes) that updates all 4 charts simultaneously.

Style: Dark theme (#0f1117 background), neon accent (#00d4ff), 
sharp geometric design language, Rajdhani or similar industrial font from Google Fonts.
No rounded corners. Grid-based layout.
