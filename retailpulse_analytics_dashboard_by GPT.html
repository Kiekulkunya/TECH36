Data Source

This dashboard uses the retail behavior dataset from Kaggle:

Retail Intelligence 100K User Behavior Dataset — retail_user_behavior_100k.csv

The dataset tracks event-level retail user behavior, including sessions, user actions, product categories, region, traffic source, device type, conversion flag, drop-off flag, price, and engagement metrics.

Short reflection

What I found most interesting was how a simple event-level retail dataset can turn into a full analytical product. The data starts as user behavior logs, but once actions, categories, regions, devices, and traffic sources are connected, it becomes a story about how customers move through a retail journey.

I learned that dashboard design is not only about charts. It is about choosing the right sequence of questions: who visits, what they do, where they drop, and which segments convert better. The funnel makes the journey easy to understand, while the matrix and stacked bar chart reveal differences across markets, products, traffic sources, and devices.

One surprise was how important defensive coding is. A single missing aggregate object or external library problem can break the whole dashboard. The final version is self-contained and includes diagnostics so errors are easier to catch and explain.
